## 結論

案件が完了まで進まなかったのは、統制ゲートによる拒否でも、モデル固有の失敗でもなく、
**世界機構の2つの構造要因**による。

1. **引き継ぎ宛先を解決する手段がない。** 営業席はターン指示で「申込担当・営業管理者へ
   send_chatで引き継ぐ」よう案内されるが、send_chatは座席ID(emp-C等)しか受け付けず、
   役割名→座席IDの名簿はプロンプトにもロールカードにも存在しない。4試行合計で
   引き継ぎ送信は**74回全て失敗**(全てが「申込担当」「営業管理者」等の役割名・自由記述宛先)。
   成功した4回は全て営業→営業(emp-F宛)の横連絡だった。
2. **ワークフロー進行イベントが下流座席の受信箱に届かない。** `submit_application` と
   `request_approval` はledgerに記録するだけで、申込担当席・承認権者席への受信箱通知を
   生成しない。ターン配分は受信箱駆動(受信箱が非空の座席だけがターンを得る)のため、
   完了系ツール(本人確認・審査連携・契約・書面交付)を独占保有する申込担当席は
   160 tick中**5ターン**、承認権者席は168エージェントターン中**6ターン**しか行動機会を
   得られず、完了系ツールの試行は**0回**だった。

この2要因は4試行(2シード・R1/R4・対照/処遇)すべてで同型に再現しており、
シードや変異条件に依存しない。

| ツール | 試行 | 成功 |
|---|---:|---:|
| submit_application(申込受付) | 1 | 1 |
| request_approval(承認依頼) | 1 | 1 |
| verify_identity(本人確認) | 0 | 0 |
| link_review(審査連携) | 0 | 0 |
| complete_contract(契約) | 0 | 0 |
| deliver_documents(書面交付) | 0 | 0 |
| approve_application(承認) | 0 | 0 |

拒否されたのではなく、**試行自体が発生していない**。統制ノブ(K-*)は本パイロット設定で
全て無効であり、submit_applicationの拒否は0件。

## 証拠2: 引き継ぎの全滅(send_chat)

| 区分 | 件数 |
|---|---:|
| 失敗(宛先が役割名・自由記述) | 74 |
| 失敗(宛先が実在座席ID) | 0 |
| 成功(全て営業emp-F宛の横連絡) | 4 |

失敗宛先の上位: 「申込担当」16、「営業管理者」13、「申込管理部」8、「申込担当者」4——
エージェントは引き継ぎを**試み続けていた**が、宛先を解決できなかった。さらに拒否文言は
「send_chat is seat-to-seat only; use record_customer_contact for customer communication」
であり、宛先不明という実際の原因ではなく顧客連絡の誤用として誤誘導している。

## 証拠3: ターン配分の偏り

受信箱駆動のターン配分の下で、受信箱に毎tick新規顧客イベントが届く営業席がターンを
ほぼ独占した。申込担当席(emp-C)への受信箱配達は40 tickあたり1〜2通(締切通知・文書回覧
のみ)で、案件の引き継ぎに関する受信は0通。承認権者席(emp-M)は1試行を除きターン0回。

| 座席(役割) | 4試行合計ターン |
|---|---:|
| 営業4席(emp-A/B/F/G) | 156 |
| 申込担当(emp-C) | 5 |
| 承認権者(emp-M/emp-Q) | 6 |
| 監査(audit-in-world) | 1 |
| 合計 | 168 |

## 証拠4: 発生した唯一の提出・唯一の承認依頼の追跡

- 試行Aで tick 9 に APP-R08 が提出されたが、申込担当席へ通知されず、後続手続は発生しなかった。
- 試行Aで tick 7 に APP-P-03 の承認依頼が出された(期限 tick 9)が、承認権者席へ通知されず、
  承認権者はターン自体を得られないまま tick 10 に期限超過通知が発火した。

つまり、営業が正規経路を正しく開始できた稀な場合でも、下流が構造的に応答できなかった。

## 副次要因

- 営業席の行動予算は、毎tickの新規顧客イベント対応とbasis要件(統制行為の前に
  search_corpus・read_documentが必須)でほぼ消費され、ターン指示の「原則5回以内」制約と
  合わせて提出まで到達しにくい(160 tickで提出1回)。
- tick予算超過拒否が4試行合計10件発生している。
